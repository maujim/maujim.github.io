# lego-hand-grid.txt

Purpose: architecture notes for `projects/lego-hand-grid.html`.

This file explains the moving parts quickly so future edits can be targeted.

---

## 0) What this version does

Three.js + MediaPipe HandLandmarker LEGO prototype with:
- 16x16 stud grid,
- hand pinch pick-up / drag / drop,
- selection visual hierarchy,
- **infinite red side supply**:
  - one supply spawn spot on the left,
  - a polling loop checks every 1s,
  - if spot is empty, it spawns a new red 2x3 brick.

This allows repeatedly grabbing bricks and placing them on the board.

## 2) Important constants

Grid/brick:
- `GRID_COLS`, `GRID_ROWS`, `BRICK_W`, `BRICK_D`

Side supply:
- `PILE_SPAWN`: world position of supply spot
- `PILE_OCCUPY_RADIUS`: radius used to decide if spawn spot is occupied
- `PILE_SPAWN_POLL_MS`: poll interval (1000 ms)

Pinch tuning:
- base thresholds: `0.43 / 0.58`
- scale: `PINCH_RING_SCALE = 1.09`
- effective thresholds:
  - on: `0.43 * 1.09`
  - off: `0.58 * 1.09`
- debounce frames:
  - pinch-on requires 2 frames
  - pinch-off requires 3 frames

## 3) Brick model (multi-brick)

Bricks are no longer single-instance.

- `bricks[]`: array of brick data objects
- each brick data has:
  - `id`
  - `group` (Three.js group)
  - placement mode:
    - `onGrid` with `cornerI/cornerJ`
    - or side/world placement with `worldX/worldZ`

Key functions:
- `createBrickInstance(...)`
- `getBrickCenterXZ(brickData)`
- `syncBrickPose(brickData, lift)`
- `syncAllBrickPoses()`

---

## 4) Infinite side supply behavior

Supply spawn loop:
- `isPileSpotEmpty()` checks if any brick center is within `PILE_OCCUPY_RADIUS` of `PILE_SPAWN`.
- `maybeSpawnSupplyBrick()` spawns one red brick at the supply spot if empty.
- called at startup once, then via:
  - `setInterval(maybeSpawnSupplyBrick, PILE_SPAWN_POLL_MS)`

Effect:
- move a supply brick away,
- spot becomes empty,
- next poll (~1s) creates another brick.

---

## 5) Selection / drag visual hierarchy

Shared decorators move between brick groups:
- `hoverOutline` (faint)
- `selectedOutline` (strong)
- `gizmo` (axes)

`updateSelectionVisuals()`:
- binds decorators to hovered/selected brick,
- applies subtle emissive tint per brick:
  - selected > hovered > none.

Ghost preview:
- `ghostBrick` shown only while dragging,
- cyan when valid, red when invalid (`setGhostColor`).

---

## 6) Hand input state machine

Main loop: `updateFromHand()`

Per frame with new video time:
1. detect landmarks with `detectForVideo`
2. compute pinch metric = thumb-index distance / palm width
3. apply hysteresis + debounce to update `pinchDown`
4. compute cursor midpoint and raycast onto ground plane
5. update hovered brick from:
   - pickable mesh raycast first,
   - then footprint tolerance fallback

Transitions:
- pinch start:
  - if hovering a brick → start dragging that brick
- while pinching + dragging:
  - update ghost target corner and move held brick above target
- pinch end:
  - if valid target → commit to grid corner
  - else restore from drag start state
  - always clear selection

No-hand fallback:
- clears hover/pinch states,
- cancels active drag (restores start pose),
- hides pointer + ghost.

---

## 7) Drag placement logic

- target corner from `worldToCornerRaw(world.x, world.z)`
- validity from `isValidCorner(i, j)`
- invalid preview uses clamped corner (`worldToCornerClamped`) and red ghost
- commit sets dragged brick to `onGrid=true` + target corner

---

## 8) Hover detection details

`updateHoverFromPointer()`:
1. raycast all brick pickables (precise)
2. fallback: footprint tolerance (`pointInsideBrickXZ(..., 0.28)`) for forgiving selection

## 10) Known simplifications (v0)

- No collision checks between placed bricks yet (stack/overlap prevention missing).
- Only one hand is used (`numHands: 1`).
- Drag validity checks only board bounds (not occupancy).
- Supply spawn uses polling (`setInterval`) by design.

---

## 11) Good next tuning knobs

- Increase/decrease `PILE_OCCUPY_RADIUS` if supply re-spawns too eagerly/slowly.
- Adjust pinch debounce for responsiveness vs stability.
- Add occupancy rules if overlapping bricks becomes an issue.
- Add brick rotation controls (gesture/button).
