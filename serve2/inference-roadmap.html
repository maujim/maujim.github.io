Yes. That is basically it.

Text becomes IDs. IDs become vectors. Sixteen layers repeatedly rewrite those vectors. A final norm and tied output projection produce one score for every vocabulary token. You select one, append it, and repeat. The work is getting every shape, axis, operation, and cache exactly right.

Your working syllabus

Build in dependency order.

Each stage ends at a boundary you can compare with a trusted MLX or Transformers run. Checkboxes persist locally.

Freeze a reference

Record the checkpoint revision. Inspect config, tokenizer, safetensors metadata, names, and shapes. Save reference token IDs, intermediate activations, logits, and one greedy output.

DONE WHENYou can explain every model file and reproduce a known run outside your engine.

Implement byte-level BPE

Replace ID 0 with special-token handling, regex pre-tokenization, UTF-8 byte mapping, BPE merging, vocabulary lookup, and reverse decoding.

DONE WHENExact IDs match for ASCII, whitespace, punctuation, Unicode, and special tokens; decoding round-trips.

Read tensors for real

Parse the 8-byte safetensors header length, JSON metadata, and raw ranges. Create a non-owning tensor view. Implement exact BF16→FP32 conversion. Map the file or read ranges instead of copying 709 MB.

DONE WHENYou can request any tensor by name and match several values with a reference.

Make tiny FP32 math

Write transparent loops for matmul, add, multiply, gather, RMSNorm, SiLU, softmax, reshape/view, and causal depthwise convolution.

DONE WHENEvery primitive passes a hand-computed test and randomized reference comparison.

Gather input embeddings

For each token ID, read one 1,024-value BF16 row from model.embed_tokens.weight and convert it for computation.

DONE WHENKnown IDs produce the reference [S, 1,024] activation.

Build the shared SwiGLU branch

All 16 layers use w2(silu(w1(x)) × w3(x)) with an effective intermediate width of 4,608, preceded by RMSNorm and followed by a residual add.

DONE WHENOne normalized FFN branch matches the reference.

Finish convolution layer 0

Normalize; project 1,024→3,072; split B, C, x; form B×x; causal depthwise-convolve width 3; gate with C; output-project; add residual; run FFN.

DONE WHENLayer 0 matches for a short sequence without caching.

Finish attention layer 2

Build 16 Q heads and 8 K/V heads of width 64. Apply Q/K RMSNorm, RoPE, a causal mask, scaled dot products, softmax, 2:1 grouped-query sharing, output projection, residual, and FFN.

DONE WHENQ, K, probabilities, operator output, and layer output all match.

Run the 16-layer stack

Construct operators from layer_types. Keep batch 1, short contexts, and no cache. Feed each [S, 1,024] output directly to the next layer.

DONE WHENActivations match after layers 0 through 15.

Predict one correct token

Apply final RMSNorm. Multiply the last hidden vector by the tied embedding matrix transpose. Compare 65,536 logits and choose argmax.

DONE WHENYour top logits and greedy next token match exactly enough to explain any numerical difference.

Cache, template, stream

Separate prefill from decode. Cache attention K/V and the last three convolution states. Add the chat template, EOS handling, greedy repetition, and incremental text decoding.

DONE WHENA formatted prompt generates matching greedy text without recomputing its prefix.
First component

Text becomes integers.

The model never sees words or characters. The checkpoint's tokenizer is byte-level BPE: a reversible byte representation compressed into frequently learned pieces.

1. Protect special tokens

Recognize control strings before ordinary splitting.

2. Regex pre-tokenize

Isolate contractions, letters, short number runs, punctuation, and whitespace patterns.

3. Map UTF-8 bytes

Use the reversible byte-level alphabet, ensuring arbitrary input is representable.

4. Apply BPE ranks

Repeatedly merge the best-ranked adjacent pair until no merge remains.

5. Lookup and reverse

Map pieces to IDs; reverse the process to decode generated IDs.

The model core

Ten local mixers. Six global mixers.

LFM2.5 is hybrid. Convolution cheaply mixes nearby history; attention reaches across the full causal context. Every layer wraps its mixer with normalization, residuals, and the same SwiGLU feed-forward branch.

Close the loop

One vector votes on every token.

After layer 15, normalize the last position and score every vocabulary entry. You do not need softmax for greedy argmax because it preserves ranking.

The learning method

Find the first wrong number.

Never debug 354 million parameters at once. Compare boundaries from the front; the first mismatch points to the operation that caused it.

Use the reference as a microscope.

Batch 1. Two to five tokens. No sampling. Compare token IDs → embeddings → operator branch → FFN branch → each layer → final norm → top logits. Track maximum absolute and relative error rather than assuming fused BF16/FP32 paths will be bit-identical.

Debug in this order

  1. Shape
  2. Axis
  3. Transpose
  4. Operation order
  5. Dtype
  6. Numerical tolerance
Guard the curriculum

Minimal means correct before fast.

Build now

  • Batch size 1
  • BF16 weight reading
  • FP32 activation math
  • Simple, obvious loops
  • Greedy argmax
  • Short prompts
  • Exact LFM2.5 architecture
  • Caches after uncached correctness

Build later

  • Batch scheduling
  • SIMD, threads, Metal, GPU
  • Quantized formats
  • Top-k, top-p, penalties
  • Huge contexts
  • Generic model abstractions
  • Flash attention and fused kernels

The finish line is surprisingly small.

Read the checkpoint. Format and tokenize a prompt. Run the exact 16-layer forward pass. Apply final normalization and the tied output head. Select, cache, stop on EOS, and decode readable text. Everything after that is speed, flexibility, or generation policy.
